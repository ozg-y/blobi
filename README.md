# BLOBI 🩷

A one-thumb pixel-art mobile arcade game, built with **React + Vite** and packaged
for the **Apple App Store** with **Capacitor** (native iOS shell around the web game).

> Tap & hold to turn Blobi invisible and phase through a monster — but while cloaked
> the blob is blind: it stops bouncing and dies if it drifts into a wall. Time your
> cloak, slip past the monster, then release before you hit an edge.

Game rules live in [`blobi-handoff/README.md`](blobi-handoff/README.md).

---

## Tech stack

| Layer | Choice |
|---|---|
| UI shell | React 18 + Vite |
| Game | Framework-free `<canvas>` engine ([`src/game/blobiGame.js`](src/game/blobiGame.js)) |
| Native wrapper | Capacitor 6 (iOS / WKWebView) → submittable to the App Store |

The game itself is a single self-contained engine class that runs its own
`requestAnimationFrame` loop and renders into a low-res offscreen buffer that is
blitted with nearest-neighbor scaling for crisp pixel art. React only mounts it.

## Run it in the browser (dev)

```bash
npm install
npm run dev
```

Open the printed `localhost` URL. Click & hold (mouse) = cloak. Works on desktop
for testing; use a phone / device-emulation in DevTools for the real feel.

```bash
npm run build     # production build → dist/
npm run preview   # preview the production build
```

## Ship it to the App Store (do this on a Mac with Xcode)

Everything above works on any OS. The steps below **require macOS + Xcode**
(Apple's toolchain is Mac-only) and an **Apple Developer Program** membership
($99/yr) to submit.

### 1. One-time: add the iOS platform

```bash
npm install
npm run build
npx cap add ios          # generates the native ios/ project (only once)
```

### 2. Every time you change the game

```bash
npm run ios:open         # = build + cap sync ios + cap open ios
```

This rebuilds the web assets, copies them into the native project, and opens
**Xcode**.

### 3. In Xcode — configure the app

- Select the **App** target → **Signing & Capabilities**:
  - Check **Automatically manage signing**.
  - Pick your **Team** (your Apple Developer account).
  - The **Bundle Identifier** is `com.mdpgroup.blobi` (change it in
    [`capacitor.config.json`](capacitor.config.json) if you want a different one,
    then re-run `npx cap sync ios`).
- Set **Deployment Info → iPhone Orientation** to **Portrait only** (this game is
  portrait). Uncheck landscape options.
- Add your **App Icon** and **Launch Screen** (see "Assets" below).

### 4. Test on a device / simulator

- Pick a simulator or a plugged-in iPhone in the Xcode toolbar → **▶ Run**.

### 5. Archive & upload

- Toolbar device selector → **Any iOS Device (arm64)**.
- Menu **Product → Archive**.
- In the Organizer window that opens → **Distribute App → App Store Connect →
  Upload**.

### 6. App Store Connect

- Go to <https://appstoreconnect.apple.com>, create the app record (same bundle
  ID), attach the uploaded build, fill in screenshots / description / privacy
  answers, and submit for review.

## Assets you still need before submitting

Apple requires these — they are **not** generated by this repo:

- **App Icon** — 1024×1024 PNG (no alpha). Easiest path: install
  [`@capacitor/assets`](https://github.com/ionic-team/capacitor-assets), drop a
  `assets/icon.png` (1024²) and `assets/splash.png` (2732²), then:
  ```bash
  npm i -D @capacitor/assets
  npx capacitor-assets generate --ios
  ```
- **Screenshots** for the store listing (taken from the simulator).
- **Privacy**: this game stores only a local high score in `localStorage` and
  collects **no** data — answer App Privacy accordingly ("Data Not Collected").

## Project layout

```
index.html               Vite entry (viewport-fit=cover for full-screen)
capacitor.config.json    Capacitor / iOS config (app id, name, colors)
vite.config.js           base:'./' so assets load from the app bundle
src/
  main.jsx               React bootstrap
  App.jsx
  index.css              full-screen, no-select, no-scroll game styles
  components/
    GameCanvas.jsx        React shell: mounts the engine, wires input/resize
  game/
    blobiGame.js          the whole game (loop, blob, monster, cloak, FX, HUD)
blobi-handoff/            original spec + visual mockup
```

## Tuning

All balance values (speed, ramp, hitboxes, spawn margins, shake, etc.) live in the
`TUNING` object at the top of [`src/game/blobiGame.js`](src/game/blobiGame.js).
Tweak and reload.
